문제정의

많은 기업들이 데이터베이스 관리 시스템(DBMS : DataBase Management System) 사용하고 있습니다. 그러나 기업은 데이터베이스 시스템의 한계로 인해 정형 데이터만 처리할 수 있었고, 비정형 데이터에 대한 처리에 어려움을 겪고 있습니다. 비정형 데이터는 저장된 데이터를 불러올 때마다 구조화 과정을 거쳐야 하며, 이때 기본적으로 데이터양이 늘어나기 때문에 대부분의 기업들은 비정형 데이터 처리에 대한 비효율성을 경험하고 있습니다.

가설설정

비정형 데이터를 사용자에게 시각적으로 표현하면, 사용자들은 비정형 데이터 관리에 용이해지고, 인사이트를 추출하는 기능을 향상시킬수 있을것 같았기 때문에, ThanoSQL의 사용 유저 수가 증가할 것으로 예상했습니다.

아웃풋

1. Query Manager - Data Viewer

쿼리 매니저의 데이터 뷰어 기능은 정형 데이터와는 다른 방식으로 비정형 데이터를 표현해야 했습니다. 기존 DBMS에 존재하지 않는 기능이었고, 비정형 데이터는 정형 데이터와 달리 구조화된 데이터가 아니기 때문에 오직 경로를 통해 비정형 데이터를 확인할 수 있기 때문이고, 출처의 다양성으로 인해 구조화되지 않은 형식일 때 더 풍부하고 정확한 데이터를 얻어 올바르게 분석해 가치있는 인사이트를 발견할 수 있고 다양한 애플리케이션에 응용 할 수 있기 때문에 미디어를 어떻게 보여줄 수 있을지에 대해 고민을 많이 했습니다.

당시 MOU협약을 맺고 전달 받았던 데이터가 상담(오디오)데이터, 비디오 데이터, 이미지 데이터들이 많이 있었기 때문에 이미지, 비디오, 오디오를 시각화 해서 관리 할 수 있고, 또 데이터마다 형식이 다르기 때문에 파일 형식을 선택할 수 있도록 했습니다.

또, 기술적으로 수 많은 데이터들 중 비정형 데이터만 있는 컬럼을 고르기란 어렵고, 정확한 결과 도출을 위해 비정형 데이터가 있는 컬럼을 사용자가 직접 선택 할 수 있도록 했습니다.

마지막 결과 부분은 짧은 시간에 많은 데이터를 볼 수 있도록 배치했습니다. 해당 데이터의 파일 이름을 알아야할 뿐더러, 몇번째 row에 있는 데이터인지 확인 할 수 있어야 이와 관련된 다른 데이터도 확인 할 수 있기 때문에 같이 나타내주었고, 전체가 다 인사이트를 얻는데 도움이 될 수도 있지만, 몇몇 게이터만이 인사이트를 얻는데 도움이 될 수도 있다고 생각해 전체 다운로드와 개별 다운로드 기능을 추가했습니다.

2.Query Manager

또, 사용자들이 ThanoSQL이라는 새로운 데이터베이스 시스템 언어를 먼저 익숙하게 하기 위한 부분도 우선순위가 비교적 높았기 때문에 기존 개발자들이 친숙하게 느낄 수 있는 UX로 설계했고, UI는 사용자의 빠르고 편리한 Task 달성을 위해 좀 더 깔끔하게 구성했습니다.

3.Lab

Lab 서비스또한 마찬가지였습니다. 사용자들이 ThanoSQL이라는 새로운 데이터베이스 시스템 언어를 먼저 익숙하게 하기 위한 부분도 우선순위가 비교적 높았기 때문에 기존 개발자들이 친숙하게 느낄 수 있는 UX와 UI로 디자인 했습니다.
