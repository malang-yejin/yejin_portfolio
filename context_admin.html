Context

ThanoSQL 프로덕트 구축 중에 몇 가지 문제들이 발생했습니다. 영업 담당자들이 프로모션 코드를 생성할 때마다 개발팀에게 일일이 요청해야 했는데, 이는 불편함을 초래했습니다. 또한, 회사 내부에서는 사용자 수를 파악하고 사용자 계정을 관리하는 일, 플랜 관리와 관련된 작업, 그리고 사용자의 문의에 대응하는 일 등 다양한 이슈들이 발생했습니다. 이러한 문제들을 해결하기 위해 Admin 페이지를 제작하였습니다. Admin 페이지를 통해 영업 담당자들은 직접 프로모션 코드를 생성하고 관리할 수 있게 되었습니다. 또한, 내부에서는 사용자의 수를 확인하고 사용자 계정을 효율적으로 관리할 수 있습니다. 또한, 플랜 관리와 관련된 작업들도 Admin 페이지에서 간편하게 처리할 수 있으며, 사용자의 문의에도 신속하게 대응할 수 있었습니다. 이를 통해 ThanoSQL 프로덕트의 다양한 이슈들을 해결하고자 했습니다.

주로 테이블 형태의 디자인으로 대부분의 UX를 설계하고 UI를 디자인했습니다. 서버에서 받아오는 모든 정보를 표시해야 했기 때문에, 많은 양의 정보를 담을 수 있는 테이블 형식의 디자인을 활용했습니다.

Output

thanosql 디자인 시스템을 기반으로 디자인 했습니다.

1. Login Page

로그인 페이지 입니다. 화면 중앙으로 레이아웃을 구성해 한번에 인지할 수 있게 설계했습니다.

2. DashBoard

대시보드 페이지는 현재 가입한 사용자 수만 확인할 수 있으며, 기능 확장을 고려하여 간결하게 디자인 했습니다.

3. Accounts

AAccounts 페이지는 테이블 형식으로 디자인되었으며, 서버에서 받은 모든 데이터를 관리자로서 필터링하여 편리하게 제어할 수 있습니다. 또한, 많이 조회되는 정보를 우선순위로 설정하여 효율적인 관리를 가능하게 했습니다.

4. Workspaces

워크스페이스 페이지는 관리자가 사용자가 생성한 워크스페이스를 관리할 수 있는 페이지입니다. 페이지의 핵심은 워크스페이스의 작동 여부이며, 이를 강조하는 UI 디자인을 적용했습니다.

5. Inquiry

문의하기 페이지는 thanosql.ai의 Contact Us 페이지에서 온 문의를 관리하는 페이지입니다. Contact Us에서 수집된 모든 데이터를 받아오다 보니 컬럼이 길어져서 사용자(관리자)가 주로 확인하는 정보인 이름, 이메일, 보낸 날짜, 문의 내용을 쉽게 볼 수 있도록 컬럼을 클릭할 수 있게 디자인했습니다. 이를 통해 관리자는 중요한 정보를 빠르게 확인하고 문의에 대한 관리를 원활하게 수행할 수 있게 했습니다.

6. Payment order

요금제 관리 페이지는 사용자가 현재 사용 중인 요금제와 신청 및 갱신한 요금제를 확인할 수 있는 페이지입니다. 이 페이지는 워크스페이스의 생성 과정과 워크스페이스 생성 요청 조회가 주요 목적이므로 생성 상태와 생성 요청 상태를 강조하여 디자인했습니다. 사용자(관리자)는 페이지를 통해 요금제 정보와 생성 상태, 생성 요청 상태 등을 쉽게 파악할 수 있습니다. 이를 통해 요금제 관리와 워크스페이스 생성에 대한 효율성을 높일 수 있습니다.

7. Plan

요금제 페이지 입니다. 요금제 관리를 관리자가 쉽게 할 수 있게 디자인 했습니다.

8. Promotion

프로모션 페이지 입니다. 사용자(관리자)가 쉽게 프로모션 쿠폰을 생성할 수 있게 했습니다.

The End

해당 프로젝트는 사내에서만 사용되는 프로덕트이지만, 고객과 밀접한 관련이 있으며 관리자의 경험을 고려하여 서비스가 원활하게 운영되도록 디자인해야 한다는 점에서 더욱 신중하게 디자인 해야 했던 프로젝트였습니다.
